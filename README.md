# Atman

> 실시간 위치 기반 의료·돌봄 인력 매칭 앱
> 요양병원·중소병원 야간 간호사 시프트부터 시작해 의료/돌봄 전문직 전반으로 확장한다.

---

## 1. 무엇을 만드는가

병원·요양병원이 단기 시프트 인력이 필요할 때, 가까운 거리에 등록된 워커에게 **즉시 푸시 알림**을 보내 매칭한다. 워커는 미리 등록한 2~3개 선호 지역과 자격증·근무 형태 선호도를 기반으로 자기에게 맞는 일만 골라 받는다. 근무 시작/종료는 **QR 체크인/체크아웃**으로 자동 기록되어 시급이 자동 정산된다.

### 핵심 차별점

- **실시간 위치 기반 매칭** — 한국 헬스케어 매칭 앱에 거의 없는 영역
- **QR 체크인/체크아웃** — 출퇴근 시간 자동 기록 + 정산 자동화
- **단일 데이터 소스** — 워커·시프트·매칭·정산이 하나의 PostgreSQL 흐름
- **즉시 정산(Phase 2)** — 매칭 수수료보다 큰 캐시카우, 핀테크 파트너 협업

## 2. 1순위 타겟 — 요양병원 야간 간호사 시프트

처음부터 좁힌다. 검증 가능한 가장 작은 시장에 집중.

| 기준 | 선택 |
|---|---|
| 지역 | 수도권 (서울·경기) 5~10개 시설부터 |
| 자격증 | 간호사 + 간호조무사 |
| 시설 | 요양병원·중소병원·의원 |
| 형태 | 시프트만 (야간 우선) |
| 단가 | 야간 간호사 시급 35,000~45,000원 |
| 시프트당 매출 | 28~36만 원 → 수수료 12% = **3.4~4.3만 원 마진/건** |

### 왜 요양병원 야간 간호사인가

1. **인력난이 만성적·심각** — 24시간 운영인데 야간 간호사 부족
2. **자체 풀이 약함** — 대형 병원과 달리 인력업체 의존 적음
3. **시급이 가장 높음** — 수수료 마진 최대
4. **검증이 단순** — 시프트 단위 + 간호사 면허 하나
5. **법적 분류 단순** — 의료법, 장기요양기관 지정 불필요

### 시작 시 제외

- ❌ **요양보호사** — 장기요양기관 지정 필요 또는 가족 직접 고용(케어닥 영역). MVP 부적합.
- ❌ **한방병원** — 간호사 채용 자체가 적고 시장 작음
- ❌ **대형 병원·대학병원** — 자체 풀 강함, 의사결정 느림
- ❌ **정규직 채용** — 시프트만

→ 검증 후 Phase 2에서 확장 (아래 로드맵 참고).

## 3. 시장 컨텍스트

| 지표 | 수치 |
|---|---|
| 한국 간호사 면허 보유 | 약 50만 명 (활동 약 24만) |
| 비활동 잠재 풀 | 약 26만 명 |
| 간호조무사 | 약 70만 명 (활동 약 35만) |
| 요양병원 (1순위 타겟) | **약 1,400개** |
| 중소병원·의원 | 약 6만 개 |
| 간호사 이직률 | 13~15% (전직군 평균 2배) |

### 글로벌 검증 모델

| 앱 | 가치 | 매출 (추정) | 핵심 수익 |
|---|---|---|---|
| **ShiftKey** (US) | $2B | $750M ARR | 시프트당 fixed fee + 프리미엄 + **즉시 정산** |
| **CareRev** (US) | $500M | ~$100M ARR | 시프트 수수료 + **즉시 정산 = 캐시카우** |
| **Clipboard Health** (US) | $1.3B | $300M+ ARR | 시프트당 수수료 |

### 한국 경쟁자

| 앱 | 누적 투자 | 모델 | 약점 |
|---|---|---|---|
| 케어닥 | 700억+ | 가족↔요양보호사 매칭 | 흑자 미달성, B2C 마케팅비 큼 |
| 케어링 | 350억 | 사전 예약형 | 실시간 매칭 X |
| 데이드 | 시리즈 A 정체 | 간호사 단기 매칭 | 트랙션 부재 |
| 메디스태프 | - | B2B 인력 파견 | 사실상 인력업체 |

**시그널 해석:** 케어닥이 700억 받고도 흑자 못 낸 건 **단순 매칭 수수료로는 한국 헬스케어 시장 흑자 어렵다**는 시장 검증. 매칭 + 즉시 정산 + 보험 + SaaS 다각화가 처음부터 청사진에 들어가야 한다.

## 4. MVP 스코프 (Phase 1, 4주)

**목표:** 4주 (주말 8일) 안에 본인 폰에서 시프트 매칭 알림이 울리고 QR 체크인이 작동하는 앱.

### 포함

- ✅ 카카오 소셜 로그인 (워커 / 고용주 공통)
- ✅ 워커 프로필 + 자격증 사진 업로드 (수기 검수)
- ✅ 선호 지역 등록 (최대 3개, 카카오맵 SDK)
- ✅ 고용주: 시프트 게시 (위치, 시간, 시급, 필요 자격)
- ✅ PostGIS 반경 매칭 (Edge Function)
- ✅ Expo Push 즉시 알림
- ✅ 워커 신청 → 고용주 수락 → 매칭 확정
- ✅ **QR 체크인/체크아웃 (회전 + HMAC + 위치 검증)**
- ✅ 근무 시간 자동 계산 + 정산 미리보기

### 제외 (Phase 2 이후)

- ❌ Naver Clova OCR 자동 검증 → MVP는 사진 + 수기 검수
- ❌ PASS 본인인증 → 사업자등록 후 NICE 계약 가능
- ❌ 토스페이먼츠 실결제 → MVP는 정산 정보 표시만
- ❌ 즉시 정산 → 핀테크 파트너 확보 후 Phase 2
- ❌ 요양보호사·정규직 → 시프트만
- ❌ Next.js 어드민 웹 → Supabase Studio + SQL로 운영

## 5. 기술 스택

| 영역 | 선택 | 비고 |
|---|---|---|
| 앱 | **Expo (React Native) + TypeScript** | OTA 업데이트, EAS Build 무료 |
| 백엔드 | **Supabase** (Postgres + PostGIS + Auth + Realtime + Storage + Edge Functions) | 무료 한도로 MVP 충분 |
| 푸시 | **Expo Push Notifications** | FCM/APNs 추상화, 무료 |
| 지도 | **카카오맵 SDK** | 한국 좌표·POI 우수 |
| 인증 (MVP) | 카카오 OAuth via Supabase | PASS는 Phase 2 |
| 인증 (정식) | NICE/KCB PASS 본인인증 | 사업자등록 후 |
| 결제 (Phase 2) | 토스페이먼츠 + 에스크로 | 시프트 정산 |
| 즉시 정산 파트너 (Phase 2) | 페이워치 / 비즈머니플러스 / 토스뱅크 | 자기 자본 회피 |
| 향후 웹 | Next.js + Vercel | 어드민·B2B 대시보드 |
| 모니터링 | Sentry / PostHog | Phase 2 |

**왜 Supabase인가:** PostGIS 네이티브 지원이 결정적. Firestore는 지리 쿼리에 약하다.

## 6. 데이터 모델 (요약)

```
users          (auth.users 연결, 공통 프로필)
  ├─ workers          (워커 전용: 자격증, 선호도)
  └─ employers        (고용주 전용: 시설 정보)

worker_preferences  (워커별: 선호 지역 3개, 시급 하한, 가능 시간대)

shifts              (시프트: 위치, 시간, 시급, 필요 자격)
  └─ shift_applications  (신청 ↔ 수락)
       └─ shift_check_events  (CHECK_IN / CHECK_OUT)

license_verifications  (자격증 인증 큐, OCR 결과·수기 검수 상태)
```

### 핵심 PostGIS 쿼리

```sql
-- 새 시프트 생성 시 반경 내 워커 찾기
SELECT w.id, w.expo_push_token
FROM workers w
JOIN worker_preferences wp ON wp.worker_id = w.id
WHERE
  -- 자격증 일치
  :shift_role = ANY(w.qualifications)
  -- 선호 지역 반경 내 (PostGIS GIST 인덱스)
  AND EXISTS (
    SELECT 1 FROM jsonb_array_elements(wp.preferred_locations) loc
    WHERE ST_DWithin(
      ST_MakePoint((loc->>'lng')::float, (loc->>'lat')::float)::geography,
      :shift_location::geography,
      (loc->>'radius_m')::int
    )
  )
  -- 시급 하한 충족
  AND :shift_hourly_wage >= COALESCE(wp.min_hourly_wage, 0)
  AND wp.notification_enabled = true;
```

## 7. QR 체크인/체크아웃 (핵심 인증)

### 흐름

```
워커: 매칭 확정 → 시프트 카드 → "체크인" → 회전 QR 표시
                                              ↓
고용주: 앱 → "QR 스캔" → 카메라 → 결과 확인 → 서버 검증
                                              ↓
                      DB에 CHECK_IN 이벤트 기록 (server_ts, 위치)
                                ...
                      CHECK_OUT도 동일 흐름
                                              ↓
                              근무 시간 × 시급 자동 정산
```

### 보안 4겹

1. **회전 QR** — 30~60초마다 갱신, 스크린샷 부정 차단
2. **HMAC 서명** — 서버 비밀키로 payload 서명, QR 위조 차단
3. **위치 검증** — 스캐너 GPS와 시설 위치 비교 (반경 100m)
4. **Nonce UNIQUE** — DB 제약으로 같은 QR 두 번 처리 차단

### 페이로드 구조

```ts
{
  shiftId: string,
  workerId: string,
  action: 'CHECK_IN' | 'CHECK_OUT',
  ts: number,      // 30초마다 변경
  nonce: string,   // 1회용 UUID
  sig: string      // HMAC-SHA256(payload, QR_SECRET)
}
```

## 8. 자격증 인증 — 정황 증거 기반

**핵심 사실:** 보건복지부 면허 DB는 외부 API가 없어 **PASS와 면허번호의 직접 매칭은 불가능**. 모든 한국 헬스케어 앱이 같은 한계.

### MVP (Phase 1) — 사진 + 수기 검수

```
1. 카카오 로그인 → 실명·생년월일 확보 (간이 인증)
2. 자격증 사진 업로드 (가이드라인 박스, 자동 자르기)
3. 운영자 수기 검수 (이름·번호 패턴 확인)
4. 의심 시 반려, 위조 발견 시 신고
```

### Phase 2 — OCR + PASS 자동화

```
1. PASS 본인인증 (NICE/KCB 계약 후)
   → 실명·생년월일·CI 확보
2. 자격증 사진 OCR (Naver Clova OCR)
   → 이름·면허번호·발급일자 추출
3. 자동 매칭: (PASS 이름 + 생년월일) ↔ (OCR 이름 + 생년월일)
4. 자동 승인 / 수기 검수 큐 / 자동 반려 분기
```

### Phase 3 — 병원 실물 확인 위임

```
1차 가입: 정황 증거(이름+생년월일)로 "예비 인증"
첫 매칭 시: 병원이 실물 면허증 확인 → "정식 인증" 승급
```

### 보안 강화

- 면허증 사진: AES-256 암호화, 비공개 버킷 + 5분 서명 URL
- 접근 로그 필수
- 보관 기간: 회원 탈퇴 후 30일 자동 삭제
- 같은 면허번호 다른 사용자 등록 시 즉시 알람

## 9. 비즈니스 모델

### Phase 1 — 매칭 수수료만

```
워커: 무료 (시급 그대로 수령)
고용주: 매칭 금액의 12% 수수료
   - 시프트 1건당 평균 24~36만 원
   - 수수료 2.8~4.3만 원/건
   - 실 마진 (결제·운영비 제외): 약 1.8~3만 원/건
```

**왜 워커 무료인가:** 글로벌 표준 (ShiftKey·CareRev·Clipboard Health 모두 동일). 한국 직업안정법상 워커 수수료는 일용근로자 기준 1만 원/회 상한. 고용주 수수료는 상한 없음.

### Phase 2 — 즉시 정산 추가 (캐시카우)

**구조:**

```
[시프트 종료] → 워커 선택
   │
   ├─ 정상 정산: 고용주 결제 후 (2~4주 뒤) — 수수료 0원
   │
   └─ 즉시 정산: 핀테크 파트너 즉시 송금
       워커 수수료 1.5% 또는 정액 1,500원
       플랫폼 마진: 핀테크 파트너와 분배
```

**자금 융통:** 페이워치·비즈머니플러스 같은 임금 선지급 핀테크 협업 (자기 자본 불필요).

**효과:** 미국 통계상 워커 60~80%가 사용. ShiftKey/CareRev 매출의 30~40%가 여기서 나옴.

### Phase 3 — 부가 수익 다각화

| 수익원 | 잠재 매출 | 시점 | 비고 |
|---|---|---|---|
| 매칭 수수료 | 시프트당 ~3만 원 | 즉시 | 기본 |
| **즉시 정산** | 시프트당 1,500원 × 사용률 60% | 시프트 100건+ | **핵심** |
| 책임 보험 중개 | 보험료의 25~30% | 활성 워커 500+ | KB손보·메리츠 등 |
| B2B SaaS (병원 인력관리) | 월 15~25만 원/병원 | 활성 병원 100+ | 별도 제품 |
| 유료 보수교육 | 강의 2~3만 원 | 활성 1만+ | 의무 보수교육 8h/년 |
| 광고 | 의료기기·보험사 | 활성 5만+ | Phase 3+ |

## 10. Unit Economics

### 시프트 1건 기준

```
시급 30,000원 × 8시간 = 240,000원 (워커 임금)
                        ↓
       고용주 수수료 12% = 28,800원 (매출)
                        ↓
- 결제 수수료 (토스) ~2.5%   = -6,000원
- 산재보험 처리 비용         = -2,000원
- 인력당 운영비 (분쟁/CS)    = -3,000원
                            = 약 17,800원 (실 마진)
   실 take rate: 약 7.4%
```

야간 간호사 시급 40,000원 기준이면 마진은 약 25,000원/건.

### 손익분기점 시나리오

| 시나리오 | 월 시프트 | 활성 워커 | 활성 시설 | 월 매출 | 월 이익 |
|---|---|---|---|---|---|
| 초기 사이드 | 50 | 25 | 8 | 144만 | 60만 |
| 부업 (BEP+) | 200 | 80 | 20 | 576만 | 240만 |
| 본업 진입 | 500 | 200 | 50 | 1,440만 | 600만 |
| 직원 채용 가능 | 1,500 | 600 | 150 | 4,320만 | 1,800만 |
| 시리즈 A 가능 | 5,000 | 2,000 | 400 | 1.4억 | 6,000만 |

**부업 수준 200건/월은 6개월~1년 내 도달 가능 그림.** 본업 진입은 1~2년.

## 11. 한국 규제 정리

### 직업안정법

유료직업소개사업 신고 필수 (시도지사). 수수료 상한:

| 대상 | 워커 수수료 상한 | 고용주 수수료 |
|---|---|---|
| 일용근로자 | 임금의 4% 또는 1회당 1만 원 | **상한 없음 (자율)** |
| 일반 근로자 | 월급의 30% 한도, 1회 | **상한 없음 (자율)** |

→ **고용주 측만 수수료 받는 게 법적으로 가장 합리적.**

### 요양보호사 매칭 — 함정 정리

| 경로 | 의사·간호사 필요? | 장기요양기관 지정? | MVP 가능? |
|---|---|---|---|
| 방문요양 (재가급여) | ❌ | ✅ 필요 | 어려움 |
| 요양원 상주 | ✅ | ✅ 필요 | 어려움 |
| 요양병원 보조 | ✅ (병원) | ❌ (의료법) | 가능 |
| 개인 가족 직접 고용 | ❌ | ❌ | 가능 (케어닥 영역) |

→ **MVP에선 의료법 영역(요양병원·중소병원)만.** 요양보호사는 Phase 3에서 장기요양기관 협업 또는 가족 매칭으로.

### 의료법

매칭 인력 자격증 위조 시 의료법 위반 + 형사처벌. 약관에 명시 + 위조 신고 프로세스 + 보험 가입 권장.

### 임금 선지급 (즉시 정산)

대부업 vs 결제대행 회색지대. **핀테크 파트너와 협업 구조**로 우리는 매칭 플랫폼 책임만.

## 12. 로드맵

### Phase 1 — MVP (0~6개월)

**좁고 깊게**

- 지역: 서울·경기 일부
- 타겟: 요양병원 + 중소병원 5~10개
- 인력: 간호사 + 간호조무사
- 형태: 시프트 (야간 우선)
- 자격증 인증: 사진 + 수기 검수
- 결제: 단순 매칭 수수료만

### Phase 2 — 의료기관 확장 (6~12개월)

- 타겟 추가: 의원·방문간호센터·응급실
- 자격증 인증: PASS + Naver Clova OCR
- 결제: 토스페이먼츠 에스크로
- **즉시 정산 도입 (핀테크 파트너)**
- 책임 보험 자동 묶음

### Phase 3 — 인접 자격증 확장 (12~18개월)

| 직군 | 시급 | 시장 |
|---|---|---|
| 약사 (24h 약국·휴일) | 40,000~60,000원 | 약 10만 명 |
| 사회복지사 | 18,000~25,000원 | 약 25만 명 |
| 물리·작업치료사 | 20,000~30,000원 | 약 8만 명 |
| 응급구조사 | 25,000~32,000원 | 약 3만 명 |
| 치과위생사 | 20,000~28,000원 | 약 8만 명 |

→ 같은 매칭 인프라로 인접 직군 확장. B2B SaaS 시작.

### Phase 4 — 위치기반 다업종 (18개월~)

검증된 인프라로 비-헬스케어 진출:
- 호텔·리조트 (성수기·주말)
- 행사·이벤트 인력
- 콜센터 단기
- 식음료 시즌 알바

차별점: 헬스케어에서 가져온 **자격증 인증·평점·즉시 정산 인프라**.

## 13. 한국 시장 위협 5가지

| 위협 | 강도 | 대응 |
|---|---|---|
| **병원 자체 풀** (자체 인력으로 해결) | ★★★★★ | 자체 풀 약한 곳 타겟: 요양병원·중소 의원 |
| **직거래 이탈** (카톡으로 직접 연락) | ★★★★ | 즉시 정산·보험·평점 등 부가가치로 lock-in |
| **의료 사고 책임 분쟁** | ★★★★ | 약관 + 책임보험 자동 가입 |
| **신뢰 형성 비용** | ★★★ | 평점·후기, 초기 본인 수기 신원 확인 |
| **단가 낮음** (요양보호사 영역) | ★★★ | 간호사·약사 등 고시급 직군부터 |

## 14. 성공 확률 — 솔직한 추정

| 시나리오 | 확률 | 정의 |
|---|---|---|
| 폐업/접음 (12개월 내) | **45%** | MVP 만들고 트랙션 없음 |
| 부업 수준 | **35%** | 월 매출 50~500만 원 |
| 본업 수준 | **12%** | 월 매출 1,000~3,000만 원 |
| 시리즈 A 스케일 | **6%** | 직원 10명+ |
| 유니콘 | **<1%** | — |

**부업 진입 35%, 본업 진입 12%.** 사이드 프로젝트 통계로는 양호.

### 그래도 시도할 만한 이유

- 비용 17만 원 + 시간 8일 (MVP)
- 글로벌 검증된 모델 (ShiftKey $2B)
- Phase 1만 만들면 시장 신호 확실히 잡힘
- 실패해도 학습 가치 큼

### 멈춰야 할 신호

- 인터뷰 5명 중 4명+ "괜찮아요" → 수요 없음
- 첫 3개월 베타 시프트 30건 미만 → 양면 시장 실패
- 직거래 이탈 50%+ → 비즈니스 모델 무너짐
- 의료 사고·위조 면허 사건 → 신뢰 추락

## 15. 시작 전 검증 (가장 중요)

코드 시작 전 **토요일 하루**를 검증에 쓴다.

대상: **요양병원 야간 간호사 3명, 중소병원 간호조무사 2명** (지인 통해)

질문 3개:
1. "단기 시프트 한 번씩 뛰는 친구들 있어요? 지금 어떻게 구해요?"
2. "지도에 미리 등록한 지역에 시프트 뜨면 푸시로 알림 가는 앱이 있다면 깔겠어요?"
3. "혹시 그런 거 써본 적 있어요? 뭐가 불편했어요?"

추가: 요양병원 인사담당자 1~2명에게 같은 질문 (수요 측 검증).

**한 명이라도 "오 그거 있으면 깔지" 하면 다음 주말부터 코드 시작.**
5명 다 "지금 쓰는 거 괜찮아요" 면 멈춘다.
