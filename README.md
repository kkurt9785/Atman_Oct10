# Atman

> 실시간 위치 기반 의료·돌봄 인력 매칭 앱
> 요양병원·중소병원 야간 간호사 시프트부터 시작해 의료/돌봄 전문직 전반으로 확장한다.

> **브랜드 (2026-06-10 결정):** `Atman`은 프로젝트 코드네임. 외부 브랜드는 **법인 케셰르 / Kesher** (히브리어 "매듭·연결") + **서비스명 잇닿 / Itdat** (한국어 "잇닿다(이어져 맞닿다)" 어간).
> 도메인: `itdat.co.kr` (등록 예정)

---

## 1. 무엇을 만드는가

병원·요양병원이 단기 시프트 인력이 필요할 때, 가까운 거리에 등록된 워커에게 **즉시 푸시 알림**을 보내 매칭한다. 워커는 미리 등록한 2~3개 선호 지역과 자격증·근무 형태 선호도를 기반으로 자기에게 맞는 일만 골라 받는다. 근무 시작/종료는 **QR 체크인/체크아웃**으로 자동 기록되어 시급이 자동 정산된다.

### 핵심 차별점

- **실시간 위치 기반 매칭** — 한국 헬스케어 매칭 앱에 거의 없는 영역
- **QR 체크인/체크아웃** — 출퇴근 시간 자동 기록 + 정산 자동화
- **단일 데이터 소스** — 워커·시프트·매칭·정산이 하나의 PostgreSQL 흐름
- **즉시 정산(Phase 2)** — 매칭 수수료보다 큰 캐시카우, 핀테크 파트너 협업

## 2. 1순위 타겟 — 요양병원 야간 간호사 시프트

처음부터 좁힌다. 검증 가능한 가장 작은 시장에 집중.

| 기준 | 선택 |
|---|---|
| 지역 | 수도권 (서울·경기) 5~10개 시설부터 |
| 자격증 | 간호사 + 간호조무사 |
| 시설 | 요양병원·중소병원·의원 |
| 형태 | 시프트만 (야간 우선) |
| 단가 | 야간 간호사 시급 35,000~45,000원 |
| 시프트당 매출 | 28~36만 원 → 수수료 12% = **3.4~4.3만 원 마진/건** |

### 왜 요양병원 야간 간호사인가

1. **인력난이 만성적·심각** — 24시간 운영인데 야간 간호사 부족
2. **자체 풀이 약함** — 대형 병원과 달리 인력업체 의존 적음
3. **시급이 가장 높음** — 수수료 마진 최대
4. **검증이 단순** — 시프트 단위 + 간호사 면허 하나
5. **법적 분류 단순** — 의료법, 장기요양기관 지정 불필요

### 시작 시 제외

- ❌ **요양보호사** — 장기요양기관 지정 필요 또는 가족 직접 고용(케어닥 영역). MVP 부적합.
- ❌ **한방병원** — 간호사 채용 자체가 적고 시장 작음
- ❌ **대형 병원·대학병원** — 자체 풀 강함, 의사결정 느림
- ❌ **정규직 채용** — 시프트만

→ 검증 후 Phase 2에서 확장 (아래 로드맵 참고).

## 4. MVP 스코프 (Phase 1, 4주)

**목표:** 4주 (주말 8일) 안에 본인 폰에서 시프트 매칭 알림이 울리고 QR 체크인이 작동하는 앱.

### 포함

- ✅ 카카오 소셜 로그인 (워커 / 고용주 공통)
- ✅ 워커 프로필 + 자격증 사진 업로드 (수기 검수)
- ✅ 선호 지역 등록 (**최대 2개, 당근앱식**, 카카오맵 SDK)
- ✅ 고용주: 시프트 게시 (위치, 시간, 시급, 필요 자격)
- ✅ PostGIS 반경 매칭 (Edge Function)
- ✅ Expo Push 즉시 알림
- ✅ 워커 신청 → 고용주 수락 → 매칭 확정
- ✅ **QR 체크인/체크아웃 (회전 + HMAC + 위치 검증)**
- ✅ 근무 시간 자동 계산 + 정산 미리보기

### 제외 (Phase 2 이후)

- ❌ Naver Clova OCR 자동 검증 → MVP는 사진 + 수기 검수
- ❌ PASS 본인인증 → 사업자등록 후 NICE 계약 가능
- ❌ 토스페이먼츠 실결제 → MVP는 정산 정보 표시만
- ❌ 즉시 정산 → 핀테크 파트너 확보 후 Phase 2
- ❌ 요양보호사·정규직 → 시프트만
- ❌ Next.js 어드민 웹 → Supabase Studio + SQL로 운영

## 6. 데이터 모델 (요약)

```
users          (auth.users 연결, 공통 프로필)
  ├─ workers          (워커 전용: 자격증, 선호도)
  └─ employers        (고용주 전용: 시설 정보)

worker_preferences  (워커별: 선호 지역 2개, 시급 하한, 고용형태, 근무시간 범위)
worker_preferred_areas  (당근식 지역 핀: name, center geography, radius_m, is_active)

shifts              (시프트: 위치, 시간, 시급, 필요 자격)
  └─ shift_applications  (신청 ↔ 수락)
       └─ shift_check_events  (CHECK_IN / CHECK_OUT)

license_verifications  (자격증 인증 큐, OCR 결과·수기 검수 상태)
```

### 핵심 PostGIS 쿼리

```sql
-- 새 시프트 생성 시 반경 내 워커 찾기
SELECT w.id, w.expo_push_token
FROM workers w
JOIN worker_preferences wp ON wp.worker_id = w.id
WHERE
  -- 자격증 일치
  :shift_role = ANY(w.qualifications)
  -- 선호 지역 반경 내 (PostGIS GIST 인덱스)
  AND EXISTS (
    SELECT 1 FROM jsonb_array_elements(wp.preferred_locations) loc
    WHERE ST_DWithin(
      ST_MakePoint((loc->>'lng')::float, (loc->>'lat')::float)::geography,
      :shift_location::geography,
      (loc->>'radius_m')::int
    )
  )
  -- 시급 하한 충족
  AND :shift_hourly_wage >= COALESCE(wp.min_hourly_wage, 0)
  AND wp.notification_enabled = true;
```

### 매칭 알고리즘 — 양방향 OR (2026-06-10 결정)

워커는 두 경로 중 하나라도 만족하면 시프트 푸시를 받는다.

**방향 A. 시설 → 반경 내 워커 (옵트인, Phase 1 후반)**

```
시설이 시프트 등록
   → facilities.notification_radius_m (기본 5km) 안에
     현재 위치(workers.location)가 있는 워커 추출
   → 워커가 "현재 위치 기반 알림" 명시 동의 시에만 작동
   → 배터리·프라이버시 보호: foreground GPS만 사용 (background 추적 X)
```

**방향 B. 워커 선호 지역 → 시프트 (MVP 기본 경로)**

```
워커가 미리 등록한 worker_preferred_areas 2개 (당근앱식)
   → 시프트 위치가 워커 선호 지역 반경(기본 2km) 안에 있으면 푸시
   → 정적 데이터 (GPS 실시간 추적 불필요)
   → 출시 직후 기본 활성화
```

→ **둘은 OR 조건으로 결합.** 한 쿼리에 통합 가능.

```sql
WHERE
  ST_DWithin(w.location, :shift_location, :facility_radius)    -- A
  OR
  ST_DWithin(:shift_location, wpa.center, wpa.radius_m)         -- B
```

### 추가 필터 (워커 측 선호도)

매칭 SQL의 1차 필터로 적용 — 알림 폭주 사전 방지.

| 필터 | 컬럼 | 비고 |
|---|---|---|
| 최소 시급 | `worker_preferences.min_hourly_rate` | 기존 |
| 선호 고용형태 | `worker_preferences.preferred_employment_types TEXT[]` | `['day', 'night', 'weekend', 'short', 'long']` |
| 근무시간 범위 | `min_shift_hours`, `max_shift_hours` | 신규 |

### 운영 디자인 — 3가지 이슈 사전 결정

| 이슈 | MVP 대응 | Phase 2 검토 |
|---|---|---|
| **알림 폭주** | 매칭 결과 `LIMIT 100` + 거리순 정렬 (가까운 워커 우선) | 매칭 점수(거리+시급 적합도+평점) 상위 30명 단계 발송 |
| **프라이버시·배터리** | 방향 B(선호지역)만 출시 기본. 방향 A는 명시 동의 + foreground GPS만 | 백그라운드 위치 옵트인 (장점 안내 + 거부 자유 보장) |
| **공정성 (선착순 vs 적합도)** | 거리순 발송 — 가까운 워커가 자연스럽게 우선 | 단계 발송: 1차(상위 30명) → 5분 미응답 → 2차(다음 30명). IntelyCare 모델 차용 |

### 스키마 보강 (Phase 1 후반)

```sql
-- 시설 알림 반경 설정
ALTER TABLE facilities
  ADD COLUMN notification_radius_m INTEGER DEFAULT 5000;

-- 워커 선호 지역 정규화 테이블 (당근식)
CREATE TABLE worker_preferred_areas (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  worker_id UUID REFERENCES workers(id) ON DELETE CASCADE,
  name TEXT NOT NULL,                       -- "강남역", "역삼동"
  center geography(POINT, 4326) NOT NULL,
  radius_m INTEGER DEFAULT 2000,
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX idx_wpa_active ON worker_preferred_areas
  USING GIST (center) WHERE is_active;

-- is_active 최대 2개 제약 트리거
CREATE FUNCTION enforce_max_2_areas() RETURNS TRIGGER AS $$
BEGIN
  IF (SELECT COUNT(*) FROM worker_preferred_areas
      WHERE worker_id = NEW.worker_id AND is_active) >= 2 THEN
    RAISE EXCEPTION '선호 지역은 최대 2개까지 등록 가능합니다';
  END IF;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_max_2_areas BEFORE INSERT ON worker_preferred_areas
  FOR EACH ROW WHEN (NEW.is_active) EXECUTE FUNCTION enforce_max_2_areas();

-- 워커 필터 보강
ALTER TABLE worker_preferences
  ADD COLUMN preferred_employment_types TEXT[],
  ADD COLUMN min_shift_hours INTEGER,
  ADD COLUMN max_shift_hours INTEGER;
```

성능 영향: 기존 PostGIS PoC ~28ms(10K warm) → 양방향 OR + 필터 추가 시 ~50-80ms 예상. 임계치 100ms 이내 유지.

## 7. QR 체크인/체크아웃 (핵심 인증)

### 흐름

```
워커: 매칭 확정 → 시프트 카드 → "체크인" → 회전 QR 표시
                                              ↓
고용주: 앱 → "QR 스캔" → 카메라 → 결과 확인 → 서버 검증
                                              ↓
                      DB에 CHECK_IN 이벤트 기록 (server_ts, 위치)
                                ...
                      CHECK_OUT도 동일 흐름
                                              ↓
                              근무 시간 × 시급 자동 정산
```

### 보안 4겹

1. **회전 QR** — 30~60초마다 갱신, 스크린샷 부정 차단
2. **HMAC 서명** — 서버 비밀키로 payload 서명, QR 위조 차단
3. **위치 검증** — 스캐너 GPS와 시설 위치 비교 (반경 100m)
4. **Nonce UNIQUE** — DB 제약으로 같은 QR 두 번 처리 차단

### 페이로드 구조

```ts
{
  shiftId: string,
  workerId: string,
  action: 'CHECK_IN' | 'CHECK_OUT',
  ts: number,      // 30초마다 변경
  nonce: string,   // 1회용 UUID
  sig: string      // HMAC-SHA256(payload, QR_SECRET)
}
```

## 8. 자격증 인증 — 정황 증거 기반

**핵심 사실:** 보건복지부 면허 DB는 외부 API가 없어 **PASS와 면허번호의 직접 매칭은 불가능**. 모든 한국 헬스케어 앱이 같은 한계.

### MVP (Phase 1) — 사진 + 수기 검수

```
1. 카카오 로그인 → 실명·생년월일 확보 (간이 인증)
2. 자격증 사진 업로드 (가이드라인 박스, 자동 자르기)
3. 운영자 수기 검수 (이름·번호 패턴 확인)
4. 의심 시 반려, 위조 발견 시 신고
```

### Phase 2 — OCR + PASS 자동화

```
1. PASS 본인인증 (NICE/KCB 계약 후)
   → 실명·생년월일·CI 확보
2. 자격증 사진 OCR (Naver Clova OCR)
   → 이름·면허번호·발급일자 추출
3. 자동 매칭: (PASS 이름 + 생년월일) ↔ (OCR 이름 + 생년월일)
4. 자동 승인 / 수기 검수 큐 / 자동 반려 분기
```

### Phase 3 — 병원 실물 확인 위임

```
1차 가입: 정황 증거(이름+생년월일)로 "예비 인증"
첫 매칭 시: 병원이 실물 면허증 확인 → "정식 인증" 승급
```

### 보안 강화

- 면허증 사진: AES-256 암호화, 비공개 버킷 + 5분 서명 URL
- 접근 로그 필수
- 보관 기간: 회원 탈퇴 후 30일 자동 삭제
- 같은 면허번호 다른 사용자 등록 시 즉시 알람

## 9. 비즈니스 모델

### Phase 1 — 매칭 수수료만

```
워커: 무료 (시급 그대로 수령)
고용주: 매칭 금액의 12% 수수료
   - 시프트 1건당 평균 24~36만 원
   - 수수료 2.8~4.3만 원/건
   - 실 마진 (결제·운영비 제외): 약 1.8~3만 원/건
```

**왜 워커 무료인가:** 글로벌 표준 (ShiftKey·CareRev·Clipboard Health 모두 동일). 한국 직업안정법상 워커 수수료는 일용근로자 기준 1만 원/회 상한. 고용주 수수료는 상한 없음.

### Phase 2 — 즉시 정산 추가 (캐시카우)

**구조:**

```
[시프트 종료] → 워커 선택
   │
   ├─ 정상 정산: 고용주 결제 후 (2~4주 뒤) — 수수료 0원
   │
   └─ 즉시 정산: 핀테크 파트너 즉시 송금
       워커 수수료 1.5% 또는 정액 1,500원
       플랫폼 마진: 핀테크 파트너와 분배
```

**자금 융통:** 페이워치·비즈머니플러스 같은 임금 선지급 핀테크 협업 (자기 자본 불필요).

**효과:** 미국 통계상 워커 60~80%가 사용. ShiftKey/CareRev 매출의 30~40%가 여기서 나옴.

### Phase 3 — 부가 수익 다각화

| 수익원 | 잠재 매출 | 시점 | 비고 |
|---|---|---|---|
| 매칭 수수료 | 시프트당 ~3만 원 | 즉시 | 기본 |
| **즉시 정산** | 시프트당 1,500원 × 사용률 60% | 시프트 100건+ | **핵심** |
| 책임 보험 중개 | 보험료의 25~30% | 활성 워커 500+ | KB손보·메리츠 등 |
| B2B SaaS (병원 인력관리) | 월 15~25만 원/병원 | 활성 병원 100+ | 별도 제품 |
| 유료 보수교육 | 강의 2~3만 원 | 활성 1만+ | 의무 보수교육 8h/년 |
| 광고 | 의료기기·보험사 | 활성 5만+ | Phase 3+ |

## 10. Unit Economics

### 시프트 1건 기준

```
시급 30,000원 × 8시간 = 240,000원 (워커 임금)
                        ↓
       고용주 수수료 12% = 28,800원 (매출)
                        ↓
- 결제 수수료 (토스) ~2.5%   = -6,000원
- 산재보험 처리 비용         = -2,000원
- 인력당 운영비 (분쟁/CS)    = -3,000원
                            = 약 17,800원 (실 마진)
   실 take rate: 약 7.4%
```

야간 간호사 시급 40,000원 기준이면 마진은 약 25,000원/건.

### 손익분기점 시나리오

| 시나리오 | 월 시프트 | 활성 워커 | 활성 시설 | 월 매출 | 월 이익 |
|---|---|---|---|---|---|
| 초기 사이드 | 50 | 25 | 8 | 144만 | 60만 |
| 부업 (BEP+) | 200 | 80 | 20 | 576만 | 240만 |
| 본업 진입 | 500 | 200 | 50 | 1,440만 | 600만 |
| 직원 채용 가능 | 1,500 | 600 | 150 | 4,320만 | 1,800만 |
| 시리즈 A 가능 | 5,000 | 2,000 | 400 | 1.4억 | 6,000만 |

**부업 수준 200건/월은 6개월~1년 내 도달 가능 그림.** 본업 진입은 1~2년.

## 11. 한국 규제 정리

### 직업안정법

유료직업소개사업 신고 필수 (시도지사). 수수료 상한:

| 대상 | 워커 수수료 상한 | 고용주 수수료 |
|---|---|---|
| 일용근로자 | 임금의 4% 또는 1회당 1만 원 | **상한 없음 (자율)** |
| 일반 근로자 | 월급의 30% 한도, 1회 | **상한 없음 (자율)** |

→ **고용주 측만 수수료 받는 게 법적으로 가장 합리적.**

### 요양보호사 매칭 — 함정 정리

| 경로 | 의사·간호사 필요? | 장기요양기관 지정? | MVP 가능? |
|---|---|---|---|
| 방문요양 (재가급여) | ❌ | ✅ 필요 | 어려움 |
| 요양원 상주 | ✅ | ✅ 필요 | 어려움 |
| 요양병원 보조 | ✅ (병원) | ❌ (의료법) | 가능 |
| 개인 가족 직접 고용 | ❌ | ❌ | 가능 (케어닥 영역) |

→ **MVP에선 의료법 영역(요양병원·중소병원)만.** 요양보호사는 Phase 3에서 장기요양기관 협업 또는 가족 매칭으로.

### 의료법

매칭 인력 자격증 위조 시 의료법 위반 + 형사처벌. 약관에 명시 + 위조 신고 프로세스 + 보험 가입 권장.

### 임금 선지급 (즉시 정산)

대부업 vs 결제대행 회색지대. **핀테크 파트너와 협업 구조**로 우리는 매칭 플랫폼 책임만.

## 12. 로드맵

### Phase 1 — MVP (0~6개월)

**좁고 깊게**

- 지역: 서울·경기 일부
- 타겟: 요양병원 + 중소병원 5~10개
- 인력: 간호사 + 간호조무사
- 형태: 시프트 (야간 우선)
- 자격증 인증: 사진 + 수기 검수
- 결제: 단순 매칭 수수료만

### Phase 2 — 의료기관 확장 (6~12개월)

- 타겟 추가: 의원·방문간호센터·응급실
- 자격증 인증: PASS + Naver Clova OCR
- 결제: 토스페이먼츠 에스크로
- **즉시 정산 도입 (핀테크 파트너)**
- 책임 보험 자동 묶음

### Phase 3 — 인접 자격증 확장 (12~18개월)

| 직군 | 시급 | 시장 |
|---|---|---|
| 약사 (24h 약국·휴일) | 40,000~60,000원 | 약 10만 명 |
| 사회복지사 | 18,000~25,000원 | 약 25만 명 |
| 물리·작업치료사 | 20,000~30,000원 | 약 8만 명 |
| 응급구조사 | 25,000~32,000원 | 약 3만 명 |
| 치과위생사 | 20,000~28,000원 | 약 8만 명 |

→ 같은 매칭 인프라로 인접 직군 확장. B2B SaaS 시작.

### Phase 4 — 위치기반 다업종 (18개월~)

검증된 인프라로 비-헬스케어 진출:

- 행사·이벤트 인력

- 식음료 시즌 알바

차별점: 헬스케어에서 가져온 **자격증 인증·평점·즉시 정산 인프라**.

### 보상 지급 방식 — 현금 vs 적립 선택

워커는 보상 발생 시 둘 중 선택:

```
A. 즉시 현금       → 3만원 (카카오페이 송금)
B. 적립 (+2% 보너스, 만료 12개월)
   → 30,600원
   사용처:
   ├─ 간호용품 스토어 (메인)
   ├─ 보수교육 결제 (Phase 3)
   ├─ 보험료 결제 (Phase 3)
   └─ 현금 출금 (수수료 2%)
```

추천 보상도 동일 — 현금 2만 vs 적립 20,400원.

**예상 적립 선택률 70-80%** (토스·페이코 데이터 기준 +2% 보너스 + 만료 있는 경우). 현금 흐름 보호 효과 큼.

### 적립금 운영 룰

| 룰 | 이유 |
|---|---|
| 1회 결제에 적립금 **최대 50%까지** 사용 | 현금 매출 보호 |
| 적립금 → 다른 사용자 송금 ❌ | 환금 사기 차단 |
| 현금 출금 시 **수수료 2%** | 스토어 사용 유도 |
| 가입·첫매칭 보상은 계정당 1회 | 다중 계정 차단 |
| 추천은 휴대폰 인증 본인만 카운트 | 셀프 추천 차단 |
| 만료 30일 전 푸시 알림 | 사용 유도 |

### 차등 수수료 정책 (2026-06-12 결정)

매칭 가치에 따라 차등. 평균 실 수수료 약 12-13%.

| 시프트 유형 | 일반 수수료 | 베타 (50% 할인) | 정당화 |
|---|---|---|---|
| **응급/긴급** (24h 내 매칭 필요) | **15%** | 7.5% | "5km 반경 즉시 매칭 = 가치 ↑" |
| **일반 시프트** | **12%** | 6% | 시장 표준 (한국 인력업체 20-30% 대비 절반) |
| **정기/장기** (1주+) | **10%** | 5% | "lock-in 보상" |

**실 마진 (시급 35,000원 × 8h 기준):**
- 응급 15% → 매출 42,000 / 실 마진 ~30,000원
- 일반 12% → 매출 33,600 / 실 마진 ~21,000원
- 정기 10% → 매출 28,000 / 실 마진 ~16,000원

### 간호용품 스토어 — 드롭시핑 (MVP)

적립금 사용처의 메인 채널. 다중 수익원 동시 확보.

**운영 모델:**
```
간호용품 공급사 3-5곳 직거래 MOU
   ↓
잇닿 앱 내 스토어에서 결제 (적립금 + 카드)
   ↓
공급사 직배송 (재고·물류 우리 부담 ❌)
   ↓
우리 마진 15-25%
```

**상품 카테고리 (MVP SKU 30-50개):**

| 카테고리 | 예시 | 예상 마진 |
|---|---|---|
| 필수 의료용품 | 청진기, 펜라이트, 시력판, 압박 스타킹 | 15-25% |
| **간호 실내화** | 크록스, 모스크, 의료용 슬리퍼 | 20-30% |
| 간호복·가운 | 캐주얼 스크럽, 작업복 | 15-25% |
| 명찰·시계 | 명찰 케이스, 간호용 시계 | 25-40% |
| 야간 굿즈 | 카페인 음료, 비상 간식 | 10-20% |
| 컴포트 | 눈마사지기, 발찜질기 | 30-50% |

**예상 ARPU (월 1회 구매):** 평균 3-5만원 결제 × 20% 마진 = **6,000-10,000원/월/활성 유저**

LTV 합산: 시프트 마진 (20만) + 스토어 마진 (10년 60만+) = **70만+ LTV/간호사**.

### Unit Economics (3개월 베타 시뮬레이션)

```
비용
├─ 가입 보상 (커피): 100명 × 5천원       =  50만원
├─ 자격증 등록 보상: 80명 × 1만원        =  80만원 (가입→자격증 80%)
├─ 첫 매칭 보상: 50명 × 3만원            = 150만원 (가입→매칭 50%)
├─ 추천 보상: 30쌍 × 4만원               = 120만원 (양쪽 2만)
├─ 병원 첫 매칭 무료: 10건 × 마진 손실   =  20만원
└─ 합계                                   = 420만원

매출 (베타 50% 할인 수수료 적용)
├─ 시프트 50건 × 평균 매출 ~17,000원     =  85만원
└─ 합계                                   =  85만원

3개월 베타 net = -195만원 (CAC로 인식)
```

**지표:**
- CAC (간호사 1명 모집비): ~2만원 ⭐ (한국 SaaS 평균 5-15만원 대비 매우 낮음)
- LTV (간호사 1명 평생 가치): 70만원+ (시프트 마진 + 스토어)
- **LTV/CAC = 35배** (3배 이상이면 건강)

### 유입 채널 (한국 헬스케어 특화 Top 5)

1. **노무사·세무사 제휴** ⭐⭐⭐ (의료기관 노무 외주 70%+. 노무사 추천이 광고보다 강력)
2. **간호사 단톡방·카페 침투** (네이버 카페 "간호사 휴게실" 30만+, 추천 보상 트리거)
3. **요양병원협회 공식 제휴** (대한요양병원협회 회원 1,400개 대상)
4. **사례 콘텐츠 (블로그·유튜브)** ("월 200만원 부업 간호사 사례" SEO 장기 자산)
5. **휴직·복직 간호사 풀** (맘카페·워킹맘 카페, 시간 유연성 ↑)

광고 (페이스북·인스타·잡포털) **비추천** — 헬스케어 광고 단가 ↑, 간호사 타겟 정확도 ↓.

### 부가가치 lock-in — MVP / Phase 2 분리

**MVP (직거래 이탈 1차 방어):**
- ✅ 표준 근로계약서 자동 발행 (고용노동부 양식)
- ✅ 3.3% 원천징수 자동 정산
- ✅ 적립금 + 간호용품 스토어

**Phase 2 (직거래 이탈 90% 차단):**
- ⏳ 산재·고용보험 자동 가입 (노무사 파트너 모델)
- ⏳ 즉시 정산 (핀테크 파트너, 시급 1.5% 마진)
- ⏳ 책임 보험 단체 가입

영업 메시지 (병원 대상):
> "야간 간호사 1명 구하시면:
> ① 즉시 매칭 (반경 5km)
> ② 표준 근로계약서 자동 발행
> ③ 3.3% 원천징수 자동 처리
> ④ Phase 2부터 4대보험 신고 대행 + 사고 시 산재 자동 처리
>
> 인사담당자 일 80% 줄여드립니다. 첫 매칭 무료, 베타 3개월 수수료 50% 할인."
