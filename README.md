# Atman / 잇닿

병원이 의료 인력 공고를 게시하고 지원자를 직접 검토·채용한 뒤, 신규 인력과 기존 직원을 함께 관리하는 병원 인력운영 SaaS입니다.

## 현재 사업모델

- 직업정보 제공: 워커가 공고를 직접 탐색·선택·지원합니다.
- 병원 직접 채용: 병원이 지원자를 직접 검토하고 채용 여부를 결정합니다.
- 병원 직접 임금 지급: 잇닿은 임금을 수취·보관·재지급하지 않습니다.
- 정액 SaaS: 매출은 병원용 공고·근태·휴가·급여 검토·인력풀·관리자 기능 이용료입니다.
- 임금·채용 성공액 연동 수수료는 받지 않습니다.

직업정보제공사업의 기본 원칙에 맞춰 특정 구인자와 구직자의 고용계약 성립을 잇닿이 알선하지 않습니다. 조건 기반 공고 노출과 알림은 정보 탐색을 돕는 기능이며, 지원과 채용 결정은 당사자가 직접 수행합니다.

## 제품 구성

| 영역 | 주요 기능 |
|---|---|
| 워커 | 공고 탐색, 직접 지원, 지원 현황, 병원 QR 출퇴근, 휴가 신청 |
| 병원 | 공고 게시, 지원자 검토, 직원 등록, 근태·휴가 관리, 급여 지급 검토 |
| 공통 데이터 | 공고 → 지원 → 병원 승인 → 근무기록 → 병원 직접 지급 |

## 요금 원칙

- Free 파일럿: 직원 3명, 공고 월 3건
- Clinic: 월 59,000원, 직원 10명, 공고 월 3건, 근태·휴가·급여 검토
- Basic/Pro/Enterprise: 직원·공고·인력풀·관리자·운영 자동화 범위에 따라 확장

서비스 이용료는 워커 임금과 별도 청구하며 부가세는 별도입니다.

## 운영 원칙

- 급여 화면의 금액은 병원이 입력한 급여 기준과 근태기록을 바탕으로 한 세전 예상·검토 자료입니다.
- 세금, 4대보험, 법정수당, 일할계산은 병원이 노무·세무 기준에 따라 최종 확인합니다.
- 전체 계좌번호는 직접 등록 직원 테이블에 저장하지 않고 지급 확인용 은행명과 끝 4자리만 관리합니다.
- 면허·자격 상태는 병원의 지원자 검토를 돕는 정보이며 최종 확인과 채용 책임은 병원에 있습니다.

## 기술 구성

- `apps/admin-web`: 병원 관리자 PWA
- `apps/worker-web`: 워커 PWA
- `supabase`: PostgreSQL, PostGIS, Auth, Storage, RPC
- Vercel: `admin.itdot.co.kr`, `itdot.co.kr`
